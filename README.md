This project consists of a complete Blog Management System built with:

Backend: Node.js, Express.js, TypeScript

Database: MongoDB (Mongoose ORM)

Frontend: React.js, TypeScript

UI Library: Material UI (MUI)
Start Scripts

Backend - npm run dev
Frontend - npm start

Both backend and frontend are fully implemented with proper validations, error handling, and a clean folder structure.

Level 1 

Created all the Backend API that is listed 
1) Create API - POST - /api/blogs
2) Get all blogs API (Filter and pagination included)- POST - /api/blogs/search
3) View Blog - GET - /api/blogs/:id
4) Update Blog - PUT - /api/blogs/:id
5) Delete Blog - DELETE - /api/blogs/:id

Fully typed controllers and routes using TypeScript

Input validation using Zod schemas

Error handling with structured & consistent JSON responses

Clean separation of controllers, routes, models, validation, config, and helpers

LEVEL 2 

Created frontend components 
1) Navbar for easy navigation
2) Home page contains list of blogs 
3) Create Blog
4) Update Blog
5) Delete Blog confirmation dialog box 

Search blogs

Sort functionality

Stateful pagination

Loading states & error handling

Reusable components and API layer


LEVEL 4

1) Validated all incoming data server-side using zod schema.
2) Use of parameterized queries or ORM-safe methods has also implementd in mongo db queries.
3) Implemented Api-limiting using express-rate-limit package.
4) Logger files has also implemented.
5) Every API response returns a proper HTTP status code.
6) CORS configured correctly for frontend communication.
7) .env file used for environment variables.

The project delivers a production-ready full-stack blog application with:

✔ Strong backend architecture
✔ Clean React frontend with TypeScript
✔ Pagination + search + sort
✔ Server-side validation
✔ Advanced security enhancements
✔ Logging + rate limiting
✔ Professional error handling
